## Hypotheses

1. [REFUTED] 모바일 미디어 쿼리의 `width: min(100% - 24px, ...)` 문법이 무효여서 데스크톱 폭이 남는다. 구분 증거: 390px에서 `.app-shell`의 계산 폭이 뷰포트보다 크다. 수정: `calc` 사용.
2. [CONFIRMED] 단일 `1fr` 그리드 트랙의 자동 최소 크기와 하위 그리드 항목의 `min-width: auto`가 긴 세션 텍스트의 최소 콘텐츠 폭을 보존한다. 구분 증거: 390px에서 `.session-card` 계산 폭이 `1370.23px`이고, 런타임 오버라이드로 `minmax(0, 1fr)`와 하위 `min-width: 0`을 주면 `scrollWidth`가 `1424`에서 `390`으로 바뀐다. 수정: 축소 규칙.
3. [REFUTED] 런타임 패널의 그리드 최소 폭 또는 sticky 규칙이 모바일 미디어 쿼리 뒤에도 적용된다. 구분 증거: 오버플로 목록의 첫 원인은 `.session-section`과 `.session-card`였고, 패널은 원인 목록에 없었다. 수정: 없음.
4. [CONFIRMED] 데스크톱의 `.runtime-panel { position: sticky }`가 사용자가 말한 항상 떠 있는 동작의 직접 원인이다. 구분 증거: 계산된 `position`이 `sticky`이고 스크롤 전후 패널 상단 좌표가 뷰포트에 고정된다. 수정: static 흐름 배치.
5. [REFUTED] 데스크톱 세션 열의 기본 최소 폭이 우측 런타임 열까지 침범해, 패널이 z-index 문제처럼 보이게 한다. 구분 증거: `.session-section.right=947.453125`, `.runtime-panel.left=967.453125`로 20px 간격이 유지됐다. 수정: 없음.
6. [REFUTED] 실제 `z-index` 또는 새로운 스택 컨텍스트가 런타임 패널을 다른 UI 위에 그린다. 구분 증거: 계산된 `z-index="auto"`. 수정: 없음.
7. [CONFIRMED] 현재 세션의 활동 텍스트가 `white-space: nowrap`으로 유지돼 긴 롤아웃 메시지가 카드 내부에서 1줄 최소 폭을 만들고 시각적으로 잘린다. 구분 증거: 실데이터 `.activity-text`의 `scrollWidth=1277`, `clientWidth=788`, `whiteSpace="nowrap"`; 런타임에서 2줄 클램프와 `overflow-wrap: anywhere`를 주면 오버플로 항목 수가 `5`에서 `0`이 됐다. 수정: 두 줄 클램프와 강제 줄바꿈.
8. [CONFIRMED] E2E가 접은 뒤에도 기존 이름의 `접기` 버튼을 찾으면서 자동 새로 고침 뒤의 요소를 읽어 테스트만 실패한다. 구분 증거: 클릭 직후 `펼치기` 버튼이 존재하고 5초 뒤에만 `접기`로 돌아온다. 수정: 테스트 locator 변경.
9. [CONFIRMED] 자동 새로 고침이 `state.expanded`에 없는 세션을 모두 초기 펼침으로 되돌린다. 구분 증거: 클릭 직후 접힌 트리가 5초 뒤 다시 펼쳐진다. 수정: 최초 수신 세션만 펼침 초기화.
10. [REFUTED] 클릭 핸들러가 잘못된 노드 ID를 변경해 즉시 접힘이 적용되지 않는다. 구분 증거: 클릭 직후 `펼치기` 버튼 수가 `1`이다. 수정: 없음.

## Findings

- 390px Playwright 런타임 값: `scrollWidth=1424`, `clientWidth=390`.
- 1440px 및 390px 스크린샷 생성, 브라우저 콘솔 오류는 `[]`.
- 390px 미디어 값: `innerWidth=390`, `(max-width: 820px)=true`, `(max-width: 580px)=true`, `.workspace-grid gridTemplateColumns="1412.23px"`.
- 오버플로 핵심 값: `.session-section width="1412.23px"`, `.session-card width="1370.23px"`.
- 런타임 CSS 검증: `grid-template-columns: minmax(0, 1fr)` 및 축소 가능한 하위 항목을 함께 적용한 뒤 `scrollWidth=390`, `clientWidth=390`, `.session-card width="324px"`.
- 데스크톱 스크롤 값: 스크롤 전 `.runtime-panel.top=394.0625`, `position="sticky"`; `scrollY=850` 뒤 `.runtime-panel.top=20`.
- 데스크톱 열 경계: `.session-section.right=947.453125`, `.runtime-panel.left=967.453125`, `z-index="auto"`.
- 실데이터 활동 텍스트: `.activity-text scrollWidth=1277`, `clientWidth=788`, `overflow="hidden"`, `whiteSpace="nowrap"`.
- 활동 텍스트 토글: 클램프 전 `maxScrollWidth=1277`, `overflowing=5`; 클램프 뒤 `maxScrollWidth=821`, `overflowing=0`.
- 접힘 재현: 클릭 직후 `collapsed=1`, `expanded=1`; 5.3초 자동 새로 고침 뒤 `collapsed=0`, `expanded=2`.

## Root cause (confirmed 2026-07-10, 접힘 상태)

- Mechanism: `load()`은 수신한 모든 루트 세션에 대해 `state.expanded`에 ID가 없으면 다시 추가한다. 사용자가 접기 동작으로 ID를 제거하면, 다음 5초 자동 새로 고침이 그것을 신규 세션으로 오인해 다시 펼친다.
- Evidence: `public/app.js`의 `load()` 루프와 실브라우저 5.3초 재현.
- Toggle proof: 루트 ID를 다시 추가하는 기존 동작이 있을 때만 5.3초 뒤 접힘이 사라진다. 최초 관측 세션에만 추가하도록 제한하면 접힘이 유지돼야 한다.
- Fix scope: `public/app.js`, `test/e2e.test.mjs`.

## Root cause (confirmed 2026-07-10)

- Mechanism: 820px 이하에서 `.workspace-grid`는 단일 `1fr` 트랙을 사용한다. `1fr`의 자동 최소 크기와 그 안의 `.session-card` 및 트리 그리드 항목의 기본 `min-width: auto`가 긴 세션 제목과 활동 텍스트의 최소 콘텐츠 폭을 보존해, 366px 컨테이너 안에서 1370px 카드가 생성됐다. 그 결과 문서 본문이 1424px까지 확장됐다.
- Evidence: `/tmp/debug-codex-session-monitor-mobile-overflow.mjs`, `public/styles.css`의 820px 미디어 규칙.
- Toggle proof: 런타임에서 `minmax(0, 1fr)`와 하위 `min-width: 0`을 적용하면 재현값이 `scrollWidth=390`이 되고, 페이지를 다시 불러 오버라이드를 제거하면 `scrollWidth=1424`로 돌아왔다.
- Fix scope: `public/styles.css`의 모바일 그리드 및 축소 규칙.
