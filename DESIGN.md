# Codex Session Monitor 디자인 시스템

## 0. 리서치 로그

- 내장 레퍼런스: Linear의 정밀한 다크 제품 화면을 기준으로 선택했다. 계층형 업무 정보와 상태 밀도가 높은 화면에 잘 맞고, 장식보다 정보 위계를 우선하기 때문이다.
- Lazyweb 검색: `developer tools command center dashboard`, `AI agent orchestration monitoring dashboard` 두 쿼리로 두 화면을 확인했다. 어두운 앱 셸, 짧은 지표 스트립, 단일 핵심 작업 영역, 상태 색상의 제한적 사용, 좁은 화면에서의 정보 재배치를 채택했다.
- 이미지 생성: 사용하지 않았다. 이 도구의 핵심 시각물은 실시간 세션 트리와 상태 변화이며, 비트맵 에셋은 정보 전달을 개선하지 않는다.

## 1. 제품 정체성

로컬에서만 동작하는 Codex 작업 관제 화면이다. 빠르게 스캔할 수 있는 차분한 운영 콘솔을 목표로 하며, 친근한 장식 UI보다 실제 메인 에이전트와 서브 에이전트의 계층, 최근 활동, 실행 중인 프로세스를 우선한다.

## 2. 색상 토큰

| 용도 | 토큰 | 값 |
| --- | --- | --- |
| 캔버스 | `--canvas` | `#090b10` |
| 기본 표면 | `--surface-1` | `#10141c` |
| 상승 표면 | `--surface-2` | `#171c27` |
| 행 호버 | `--surface-3` | `#202735` |
| 테두리 | `--line` | `#2b3342` |
| 기본 텍스트 | `--text` | `#f3f6fb` |
| 보조 텍스트 | `--muted` | `#a0aaba` |
| 작은 메타 텍스트 | `--faint` | `#9ca9bc` |
| 상호작용 강조 | `--accent` | `#8492ff` |
| 강조 호버 | `--accent-strong` | `#a9b3ff` |
| 강조 표면 위 텍스트 | `--on-accent` | `#111426` |
| 작업 중 | `--working` | `#65d6a6` |
| 대기 | `--waiting` | `#f2bf68` |
| 완료 | `--completed` | `#7f8ca2` |
| 오래됨 또는 오류 | `--stale` | `#ec7f83` |

상태색은 데이터 상태를 구분하는 데만 사용한다. 버튼, 링크, 포커스는 인디고 강조색을 사용한다.

## 3. 타이포그래피와 간격

- UI 글꼴: `-apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`
- 식별자와 프로세스 값: `ui-monospace, SFMono-Regular, Menlo, monospace`
- 크기: 본문 14px, 보조 정보 11/12px, 섹션 제목 16/18px, 화면 제목은 반응형 26~36px을 사용한다.
- 간격: 레이아웃과 패딩은 4px 기반의 `4 / 8 / 12 / 16 / 24 / 32 / 48` 스케일을 사용한다. 글꼴 크기와 테두리 두께처럼 시각 밀도에 직접 영향을 주는 값은 예외로 둔다.

## 4. 컴포넌트

- `AppShell`: 최대 폭 1440px의 단일 열 운영 화면.
- `MetricCard`: 수치와 짧은 설명을 분리해 보여 주는 요약 카드.
- `SessionCard`: 메인 세션의 제목, 작업 디렉터리, 관측 상태, 에이전트 트리를 담는 표면.
- `AgentNode`: 역할 배지, 상태 칩, 최근 활동, 하위 에이전트 연결선을 제공한다.
- `RuntimeRow`: 실제 Codex 프로세스의 PID, 작업 디렉터리, 경과 시간, 자원 사용량을 보여 준다. 런타임 패널은 스크롤에 고정하지 않고 문서 흐름을 따른다.
- `StatusChip`: 텍스트와 작은 색 점을 함께 제공한다. 색만으로 상태를 전달하지 않는다.
- `Button`: 36px 이상 높이, 텍스트형 동작, 명확한 포커스 링을 사용한다.

## 5. 계층과 깊이

그림자보다 표면의 명도 차와 1px 테두리로 깊이를 만든다. 메인 세션은 가장 밝은 표면, 하위 에이전트는 얇은 연결선과 한 단계 낮은 표면으로 표현한다. 카드와 빈 상태는 12px, 버튼과 행의 보조 표면은 8px, 상태 칩은 충분히 둥글게 처리한다.

## 6. 반응형 동작

- 820px 미만에서는 세션과 런타임 영역을 한 열로 쌓는다.
- 580px 미만에서는 지표 카드를 한 열로 전환하고, 프로세스 수치와 버튼을 줄바꿈한다.
- 트리는 가로 스크롤 대신 긴 경로를 슬래시 뒤에서 줄바꿈하거나 줄임표 처리하며, 전체 경로는 제목 속성으로 확인할 수 있다. 세션 제목 안의 경로도 슬래시 뒤에서만 줄바꿈한다. 한국어 문장은 어절 내부를 자르지 않고, 한 글자 관형어와 바로 뒤 명사구를 분리하지 않는다.

## 7. 모션

새로 고침 결과는 160ms의 투명도 전환만 사용한다. 레이아웃을 과도하게 움직이는 애니메이션은 쓰지 않으며, `prefers-reduced-motion`에서는 모든 전환을 제거한다.

## 8. 접근성 기준

- 명도 대비가 충분한 텍스트와 포커스 링을 유지한다.
- 상태는 색, 점, 한국어 레이블을 함께 제공한다.
- 새로 고침 결과는 `aria-live`로 알린다.
- 모든 접기·펼치기와 새로 고침 동작은 키보드로 가능하다.
- 알려진 접근성 부채는 없다.
